Portafolio

Cronometros

React / HTML / CSS

Fecha: 20/05/2020

Puedes verlo en: Sitio web en Github Pages

Consultalo en github: https://github.com/acuellarh/React-Cronometros

En este proyecto se construyó una aplicación Frontend para crear cronómetros que cuenten el tiempo que nos demoramos haciendo determinadas tareas, muy parecido a lo que hace Toggle. la aplicación se creó con create-react-app, en este proyecto se utilizaron métodos para validar el formulario, editar los campos especificos de cada cronometro, para lo cual se utilizó ciclos y el respectivo "id" de cada uno para modificar el campo apropiado, se utilizaron eventos generados a traves de los botones y manejo de estados para controlar la lógica de la aplicación.

Posts más votados

React / HTML / CSS

Fecha: 10/02/2022

Puedes verlo en: Sitio web en Netlify

Consultalo en GitHub: https://github.com/acuellarh/favorite-posts-with-hooks

Es un proyecto Frontend que permite a los usuarios votar para incrementar o disminuir la cantidad de votos de una lista de posts de un blog, adicional permite organizarlos de manera descendente o ascendente de acuerdo a la cantidad de votos. Para implementar la APP se utilizó inicialmente create-react-app, se construyó el layout con bootstrap y bootstrap-icons. Se implementó la lógica mediante Hooks (useState y UseEffect). Se estructuró la app en componentes y se implementaron props para poder compartir parámetros y funciones entre los componentes.

Expensify

RAILS / AJAX / JQUERY / DEVICE GEM / BOOTSTRAP / HTML / CSS

Fecha: 30/03/2020

Puedes verlo en: Sitio web en Heroku

Consultalo en github: https://github.com/acuellarh/expensify-rails-5

Para este proyecto se construyó una aplicación que permite administrar y visualizar en graficos los gastos de los usuarios. Es un monolitico en Rails, donde se implementarón Scopes en los módelos, filtros con AJAX para renderizar en tiempo real, se realizaron validaciones con sus respectivos mensajes de error, se hicieron pruebas con Minitest, se habilitó un Dashboard mediante el uso de las gemas Chartkink y Groupdate, se utilizó Postgresql como motor de base de Datos, Se implementó el API con HTTP Basic y Devise para luego ser desplegado en Heroku.
